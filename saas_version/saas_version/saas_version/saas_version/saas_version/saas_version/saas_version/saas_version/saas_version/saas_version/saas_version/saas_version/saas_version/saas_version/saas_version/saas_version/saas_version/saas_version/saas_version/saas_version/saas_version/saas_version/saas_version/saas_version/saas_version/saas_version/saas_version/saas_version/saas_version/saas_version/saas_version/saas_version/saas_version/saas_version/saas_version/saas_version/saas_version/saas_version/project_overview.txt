1. WHAT IS NEXUS (PMT)?
--------------------------------------------------------------------------------
Nexus (internally called "PMT" — Project Management Tool) is a full-stack
enterprise resource planning (ERP) / project management platform designed for
professional services organizations (e.g., IT services, consulting firms).

It is a monolithic Django backend (Python) + React frontend (TypeScript),
deployed via Docker Compose with PostgreSQL, Redis, and Keycloak.

The system manages the full lifecycle of projects — from client acquisition
and contracting, through project execution (tickets, timesheets, resource
allocation), to financial tracking (invoicing, payments, expenses).

It also includes a comprehensive HRMS (Human Resource Management System)
module for managing employees, attendance, leave, payroll, and compliance.


2. TECH STACK
--------------------------------------------------------------------------------

  Backend:
    - Python 3.10+ / Django 5.x + Django REST Framework
    - PostgreSQL 16 (primary database)
    - Redis (Celery message broker + caching)
    - Celery (async background task processing)
    - Keycloak 26 (OAuth2 / OpenID Connect identity provider)
    - MinIO / S3 (file storage)
    - drf-spectacular (OpenAPI/Swagger docs)
    - Simple History (audit trail)
    - django-filter (query filtering)

  Frontend:
    - React 18 + TypeScript
    - Vite 6 (build tool)
    - Ant Design 5 (UI component library)
    - TanStack Query / React Query (server state management)
    - Zustand (client state management)
    - React Router v6 (routing)
    - Axios (HTTP client)
    - @ant-design/charts (data visualizations)
    - @xyflow/react (workflow/flow chart editor)
    - TipTap (rich text editor)
    - Leaflet (map integration for attendance geo-location)

  Infrastructure:
    - Docker & Docker Compose
    - Nginx (via Whitenoise for static files)


3. ARCHITECTURE OVERVIEW
--------------------------------------------------------------------------------

  The application is split into two main codebases:

  nexus_backend/   - Django REST API
  nexus_frontend/  - React SPA

  Authentication is handled by Keycloak (external identity provider).
  The backend uses KeycloakAuthentication to validate JWT tokens from the
  frontend. Authorization is role-based (RBAC), with roles and permissions
  synced between Keycloak and the Django database.

  Background tasks (email notifications, reminders) run via Celery workers.

6. ROLE-BASED ACCESS CONTROL (RBAC)
--------------------------------------------------------------------------------

  Four default roles with predefined permission sets:

  1. Admin
     - Full access to everything (wildcard "*")

  2. HR & Admin
     - HRMS operations: employee management, attendance, leave, payroll,
       compliance
     - Own dashboard, HRMS dashboard
     - Master data (HRMS only) — view + create + update
     - Policy documents — view only
     - Role view

  3. Project Manager
     - Full project lifecycle: create, update, delete projects
     - Client management
     - Resource allocation
     - Work items & timesheets (own + approve team)
     - Finance documents view + create + update
     - Payment & AR: dashboard, contracts, invoices
     - Expenses: view, create, approve
     - All reports

  4. Employee
     - Own dashboard
     - View assigned projects
     - Work items: create, update, transition
     - Timesheets: create, update, submit own entries
     - View own allocation
     - Attendance & leave view
     - Policy documents view


7. INFRASTRUCTURE & DEPLOYMENT
--------------------------------------------------------------------------------

  docker-compose.yml orchestrates three core services:

    postgres:16-alpine   - Primary relational database
    redis:7-alpine       - Cache & Celery broker
    keycloak:26.1.4     - Identity & access management (IAM)

    (The Django backend and React frontend are built as separate Docker
     images not shown in docker-compose.yml — deployed independently.)

  File Storage:          MinIO / S3-compatible storage for uploads
  Background Jobs:       Celery workers for email, notifications, etc.
  API Documentation:     Swagger UI at /pmt/api/docs/
                         ReDoc at /pmt/api/redoc/
  Audit Trail:           django-simple-history tracks all model changes
